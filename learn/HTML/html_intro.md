# HTML 정리

## 1. HTML 개요

HTML(HyperText Markup Language)은 웹페이지의 구조와 콘텐츠를 정의하는 마크업 언어이다

1991년 팀 버너스 리(Tim Berners-Lee)가 제안한 이후, 웹의 표준이 되어 발전을 거듭해왔다

브라우저는 HTML 문서를 해석해 화면에 표시하며, 이 과정을 통해 사용자는 웹페이지와 상호작용한다

### HTML의 특징

1. **구조와 의미 부여**
    
    HTML은 문서의 구조를 정의하는 동시에 요소에 의미를 부여한다
    
    예를 들어 `<header>`는 문서의 머리말, `<nav>`는 내비게이션, `<section>`은 콘텐츠의 구분 등을 나타낸다
    
2. **레이아웃이 아닌 콘텐츠 중심**
    
    HTML은 콘텐츠가 어떤 의미를 갖는지를 나타내는 데 집중해야 한다
    
    디자인이나 배치는 주로 CSS를 통해 구현한다
    
3. **웹 표준 준수 중요성**
    
    HTML을 웹 표준에 맞춰 작성하면 다양한 브라우저와 플랫폼에서 일관성 있는 화면을 볼 수 있다
    
    접근성(Accessibility)과 검색엔진 최적화(SEO)에도 유리하다
    
4. **확장성과 호환성**
    
    HTML5가 등장하면서 오디오, 비디오, 캔버스, SVG 등의 미디어 요소를 별도 플러그인 없이 사용할 수 있게 되었다
    
    최신 브라우저 기능을 활용할 수 있어 다양한 웹 애플리케이션 개발에 용이하다
    

### HTML 문서 예시

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>HTML 소개</title>
</head>
<body>
  <header>
    <h1>HTML 개요</h1>
  </header>

  <section>
    <h2>HTML이란 무엇인가</h2>
    <p>HTML은 웹페이지의 뼈대를 구성하는 마크업 언어이다</p>
  </section>

  <footer>
    <p>Copyright © 2024</p>
  </footer>
</body>
</html>
```

1. `<!DOCTYPE html>`은 HTML5 문서임을 선언하는 구문이다
2. `<html>` 태그는 문서의 최상위 요소이며, `lang="ko"`로 문서의 언어가 한국어임을 명시한다
3. `<head>` 요소에는 문서 메타정보, 제목(`<title>`), CSS나 스크립트 참조 등이 들어간다
4. `<body>` 요소에는 실제 화면에 표시될 콘텐츠를 작성한다
5. `<header>`, `<section>`, `<footer>`는 HTML5에서 제공하는 시맨틱 태그이며, 문서를 의미 단위별로 구분한다

---

## 2. HTML 문서 구조

HTML 문서는 크게 **머리말(head)**과 **본문(body)** 부분으로 구성된다

문서의 시작에는 반드시 `<!DOCTYPE html>`을 통해 HTML5 문서임을 선언한다

### 문서의 기본 골격

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>문서 구조 예시</title>
</head>
<body>
  <h1>HTML 문서 구조</h1>
  <p>head와 body의 역할을 구분할 수 있다</p>
</body>
</html>
```

1. `<!DOCTYPE html>`: HTML5 문서임을 선언한다
2. `<html lang="ko">`: 문서 최상위 태그이며, `lang="ko"`로 문서가 한국어임을 지정한다
3. `<head>`: 문서 정보를 담는 영역이며 메타 태그, CSS·JS 로드 등을 처리한다
4. `<body>`: 실제 화면에 표시될 콘텐츠가 들어가는 영역이며, 텍스트·이미지·링크·폼 등의 모든 요소가 포함된다

### head 영역

- 문서 자체의 정보(메타데이터)를 정의하는 곳이다
- 검색엔진이나 소셜 미디어에서 문서를 인식하기 위한 정보가 주로 들어간다
- `<meta>`, `<title>`, `<link>`, `<script>` 태그 등을 포함한다

```html
<head>
  <meta charset="UTF-8">
  <meta name="description" content="HTML 문서 구조 정리">
  <meta name="keywords" content="HTML, 문서 구조, 웹개발">
  <meta name="author" content="홍길동">
  <title>문서 구조 예시</title>
  <link rel="stylesheet" href="styles.css">
  <script src="script.js"></script>
</head>
```

### body 영역

- 화면에 표시될 콘텐츠를 포함한다
- 텍스트, 이미지, 하이퍼링크, 시맨틱 태그 등을 사용해 구조를 잡는다
- `<header>`, `<main>`, `<footer>`와 같은 시맨틱 태그로 구조를 명확히 하면 검색엔진과 접근성 측면에서 이점이 있다

```html
<body>
  <header>
    <h1>HTML 문서 구조</h1>
    <nav>
      <ul>
        <li><a href="#section1">섹션 1</a></li>
        <li><a href="#section2">섹션 2</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <section id="section1">
      <h2>섹션 1</h2>
      <p>첫 번째 섹션의 내용이다</p>
    </section>

    <section id="section2">
      <h2>섹션 2</h2>
      <p>두 번째 섹션의 내용이다</p>
    </section>
  </main>

  <footer>
    <p>제작: 홍길동</p>
  </footer>
</body>
```

# HTML 핵심 태그와 요소

HTML 문서를 구성하는 데 자주 사용되는 태그와 요소를 정리한다

각 태그는 저마다의 의미(semantic)와 역할을 갖고 있으며, 올바른 사용이 중요하다

## 1. 텍스트 관련 태그

1. **제목 태그(`h1` ~ `h6`)**
    - 문서의 제목과 부제목을 표현한다
    - `h1`은 가장 중요한 제목, `h2`는 그다음 수준, …, `h6`는 가장 낮은 수준의 제목이다
    
    ```html
    <h1>가장 중요한 제목</h1>
    <h2>부제목</h2>
    <h6>가장 낮은 수준의 제목</h6
    ```
    
2. **단락 태그(`p`)**
    - 문서에서 기본적인 문단을 표시한다
    - 글을 문단별로 구분해 가독성을 높인다
    
    ```html
    <p>이 문단은 HTML 문서에서 일반 텍스트를 표시한다</p>
    ```
    
3. **줄바꿈 태그(`br`)**
    - 텍스트에서 강제 줄바꿈을 할 때 사용한다
    - 자체로 종료 태그가 없는(empty) 요소이다
    
    ```html
    안녕하세요<br>줄바꿈이 이루어진다
    ```
    
4. **수평선 태그(`hr`)**
    - 문서 내 구분선(가로줄)을 삽입한다
    - 의미상 문단을 구분하는 역할도 한다
    
    ```html
    html
    코드 복사
    <p>위 문단</p>
    <hr>
    <p>아래 문단</p>
    
    ```
    

## 2. 목록 태그

1. **순서 없는 목록(`ul`)과 목록 항목(`li`)**
    - 불릿 포인트를 사용해 항목을 나타낸다
    - 글머리 기호가 자동으로 붙는다
    
    ```html
    <ul>
      <li>사과</li>
      <li>바나나</li>
      <li>딸기</li>
    </ul>
    ```
    
2. **순서 있는 목록(`ol`)과 목록 항목(`li`)**
    - 숫자 또는 알파벳 등으로 항목에 순서가 부여된다
    
    ```html
    <ol>
      <li>첫 번째</li>
      <li>두 번째</li>
      <li>세 번째</li>
    </ol>
    ```
    

## 3. 링크와 이미지

1. **하이퍼링크(`a`)**
    - 다른 페이지나 문서, 특정 구간, 파일 등을 연결한다
    - `href` 속성에 연결할 대상의 URL을 지정한다
    
    ```html
    <a href="https://www.example.com">Example 사이트로 이동</a>
    ```
    
2. **이미지(`img`)**
    - 웹페이지에 이미지를 삽입한다
    - `src` 속성으로 이미지 파일 경로를, `alt` 속성으로 대체 텍스트를 지정한다
    
    ```html
    <img src="image.jpg" alt="이미지 설명">
    ```
    

## 4. 문장과 구분 관련 태그

1. **강조(`strong`, `em`)**
    - `strong`은 중요한 의미를 갖고 있음을 나타낸다
    - `em`은 단순히 문맥상 강조를 부여한다
    
    ```html
    <strong>이 부분은 매우 중요하다</strong>
    <em>이 부분은 강조하는 텍스트이다</em>
    ```
    
2. **인라인 코드(`code`)**
    - 코드 조각을 문장 내에 표시할 때 사용한다
    
    ```html
    <p>HTML에서 <code>&lt;body&gt;</code> 태그는 문서의 본문을 의미한다</p>
    ```
    
3. **인용(`blockquote`, `q`)**
    - `blockquote`: 긴 인용문을 표현한다
    - `q`: 문장 안에서 짧은 인용문을 표현한다
    
    ```html
    <blockquote cite="http://example.com">
      긴 인용문을 여기에 작성한다
    </blockquote>
    <p>짧은 인용: <q>짧은 인용문</q></p>
    ```

## 5. 기타 자주 쓰는 태그

1. **구역(`div`)**
    - 구역을 나누기 위한 블록 레벨 태그이다
    - 특별한 의미 없이 레이아웃이나 스타일링을 위해 사용한다
    
    ```html
    <div class="container">
      <p>이 구역은 스타일링이나 레이아웃을 위해 사용한다</p>
    </div>
    ```
    
2. **스팬(`span`)**
    - 인라인 요소로서, 문서 내에서 특정 부분을 스타일링하거나 그룹화할 때 사용한다
    - 의미가 없고, CSS를 적용하기 위한 용도로 주로 사용한다
    
    ```html
    <p>나의 <span class="highlight">중요 텍스트</span>를 강조한다</p>
    ```
    
3. **강제 공백(`&nbsp;`)**
    - 여러 개의 공백을 표현할 때 사용한다
    - HTML은 공백이 여러 개여도 브라우저상에서 하나의 공백으로 처리되므로 필요하다면 `&nbsp;`를 활용한다
    
    ```html
    <p>공백&nbsp;&nbsp;두 개 이상을 표시</p>
    ```
    

# 시맨틱 태그

시맨틱 태그(semantic tag)는 단순히 레이아웃을 구분하는 용도를 넘어, 해당 구역이 어떤 역할을 하는지를 명시적으로 알려주는 태그이다

`<div>`, `<span>` 같은 태그는 아무 의미가 없는 단순 박스 역할만 하지만, 시맨틱 태그를 사용하면 문서 구조가 더욱 명확해지고, 검색엔진과 보조기기(스크린 리더 등)에서 정보를 해석하는 데에도 유리하다

## 1. 시맨틱 태그가 중요한 이유

1. **문서 구조 파악 용이**
    
    시맨틱 태그를 사용하면 코드만 보아도 그 영역이 어떤 의미를 가지는지 직관적으로 알 수 있다
    
    유지보수와 협업이 쉬워진다
    
2. **검색엔진 최적화(SEO)**
    
    검색엔진 크롤러가 문서를 수집할 때 시맨틱 태그를 통해 더 정확하게 콘텐츠를 파악한다
    
    적절히 구조화된 문서는 검색결과 순위에도 긍정적인 영향을 미친다
    
3. **접근성 향상**
    
    시각장애인이 사용하는 스크린 리더 등에서 시맨틱 태그를 해석해 문서의 전체적인 흐름과 구성을 효율적으로 전달한다

## 2. 대표적인 시맨틱 태그

1. **`<header>`**
    - 문서나 섹션의 머리말을 나타낸다
    - 보통 로고, 사이트 제목, 내비게이션 메뉴 등이 포함된다
2. **`<nav>`**
    - 내비게이션 링크를 담는 영역이다
    - 다른 페이지로 이동하는 메뉴나 문서 내 특정 구역으로 이동시키는 링크 등을 모은다
3. **`<section>`**
    - 문서의 독립적인 주제나 섹션을 구분할 때 사용한다
    - 일반적으로 하나의 큰 주제 단위로 묶는 것을 권장한다
4. **`<article>`**
    - 독립적인 콘텐츠 블록을 나타낸다
    - 블로그 글, 뉴스 기사, 포럼 글 등 독립적으로 배포하거나 재사용할 수 있는 콘텐츠에 사용한다
5. **`<aside>`**
    - 본문과는 간접적으로 연관된 정보를 담는 영역이다
    - 사이드바나 광고, 관련 링크 등을 넣는 데 적합하다
6. **`<footer>`**
    - 문서나 섹션의 바닥글을 나타낸다
    - 보통 저작권 정보, 링크, 연락처 등을 포함한다

## 3. 시맨틱 태그 예시

```html

<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>시맨틱 태그 예시</title>
</head>
<body>
  <header>
    <h1>사이트 제목</h1>
    <nav>
      <ul>
        <li><a href="#section1">섹션 1</a></li>
        <li><a href="#section2">섹션 2</a></li>
      </ul>
    </nav>
  </header>

  <section id="section1">
    <h2>섹션 1</h2>
    <p>시맨틱 태그를 사용해 문서를 구조화한다</p>
  </section>

  <section id="section2">
    <article>
      <h3>아티클 예시</h3>
      <p>독립적인 콘텐츠 단위이다</p>
    </article>
    <aside>
      <p>추가 정보나 광고, 관련 링크를 여기에 배치한다</p>
    </aside>
  </section>

  <footer>
    <p>© 2024 My Website</p>
  </footer>
</body>
</html>
```

### 정리

1. HTML은 웹페이지의 뼈대를 구성하는 필수 언어이며, 구조와 의미 부여에 집중해야 한다
2. head와 body로 구분된 HTML 문서 구조를 이해하면 웹페이지 전반적인 흐름을 파악할 수 있다
3. 핵심 태그와 요소(h1~h6, p, ul, ol, a, img 등)를 익히면 원하는 콘텐츠를 쉽게 표현할 수 있다
4. 시맨틱 태그를 사용해 문서를 논리적으로 구분하면, 검색엔진·보조기기에 대한 접근성과 SEO를 높일 수 있다

# 5. 이미지와 미디어 다루기

웹페이지에서 시각적 콘텐츠(이미지)나 오디오, 비디오 등 다양한 미디어를 효율적으로 사용하면 사용자 경험을 크게 향상시킬 수 있다

HTML5부터는 미디어 관련 태그가 표준으로 도입되어 플러그인 없이도 다채로운 콘텐츠를 제공할 수 있다

## 1) 이미지 삽입 (`<img>`)

```html
<img src="image.jpg" alt="이미지 설명" width="200" height="100">
```

1. `src`: 이미지를 불러올 파일 경로(URL)
2. `alt`: 대체 텍스트로, 이미지를 인식할 수 없는 경우 화면에 표시되거나 스크린 리더가 읽는 정보
3. `width`, `height`: 이미지의 가로·세로 크기 지정 (단위 없는 숫자는 픽셀로 해석)

### 이미지 삽입 시 생각해볼 거

- *대체 텍스트(alt)**를 작성하자
    - 시각장애인의 스크린 리더, 이미지 로딩 실패 시 사용자에게 콘텐츠를 전달
- 이미지를 무분별하게 큰 용량으로 사용하면 페이지 로딩 속도가 저하될 수 있으므로, 적절한 크기로 최적화한다

## 2) 이미지 요소 확장: `<picture>`와 `<source>`

반응형 웹 디자인이나 다양한 해상도(레티나 디스플레이 등)에 대응하기 위해 `<picture>` 요소를 사용할 수 있다

장치·상황별로 다른 이미지를 불러와 최적화된 화면을 제공한다

```html
<picture>
  <source srcset="image-large.jpg" media="(min-width: 800px)">
  <source srcset="image-medium.jpg" media="(min-width: 500px)">
  <img src="image-small.jpg" alt="반응형 이미지">
</picture>
```

1. `<source>`는 `srcset`과 `media` 속성을 이용해 조건부 이미지를 지정
2. `<img>` 요소는 기본 이미지를 나타낸다 (fallback 용도)
3. 브라우저는 미디어 쿼리를 해석해 가장 적합한 이미지 리소스를 불러온다

## 3) 오디오 삽입 (`<audio>`)

HTML5 이전에는 오디오 재생을 위해 플러그인(플래시 등)을 사용해야 했다

HTML5 `<audio>` 태그 덕분에 기본 플레이어 UI를 제공하며, 속성을 통해 다양한 재생 옵션을 제어할 수 있다

```html
<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  브라우저가 오디오 태그를 지원하지 않는 경우 대체 텍스트를 표시한다
</audio>
```

1. `controls`: 재생·일시정지 버튼 등 기본 컨트롤을 표시
2. `<source>`: 브라우저별 지원 코덱이 다를 수 있으므로 여러 형식을 제공할 수 있다
3. `autoplay`, `loop`, `muted` 등의 속성을 활용해 자동재생, 반복재생, 음소거 등을 지정할 수 있다

## 4) 비디오 삽입 (`<video>`)

오디오와 마찬가지로 HTML5부터 `<video>` 태그가 표준으로 채택되어, 브라우저에서 직접 재생할 수 있다

```html
<video controls width="640" height="360" poster="poster.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  지원하지 않는 브라우저에서는 대체 콘텐츠를 표시한다
</video>
```

1. `controls`: 재생, 일시정지, 볼륨 조절 등의 플레이어 UI를 표시
2. `width`, `height`: 동영상의 크기를 지정한다
3. `poster`: 동영상 재생 전이나 로딩 중에 보여줄 이미지 경로
4. `autoplay`, `loop`, `muted` 속성도 오디오와 비슷한 방식으로 사용 가능

### 자막과 자막 파일(`.vtt`)

멀티미디어의 접근성을 높이기 위해 자막을 제공할 수 있다

`<track>` 태그를 이용하면 `.vtt`(WebVTT) 형식으로 자막을 작성하고 로드할 수 있다

```html
<video controls>
  <source src="video.mp4" type="video/mp4">
  <track kind="subtitles" src="subtitles.vtt" srclang="ko" label="Korean">
</video>
```

- `kind="subtitles"`: 자막 형태를 지정
- `srclang="ko"`: 자막 언어 설정
- `label="Korean"`: 자막 선택 메뉴에서 노출될 언어명

## 5) 접근성과 최적화

1. **반응형 이미지**
    - `<img>`에 `max-width: 100%` CSS를 적용해 화면 크기에 맞게 자동 조정되도록 할 수 있다
    - `<picture>` 요소로 다양한 해상도·디바이스별 이미지 제공이 가능하다
2. **미디어 포맷 호환성**
    - 브라우저마다 지원하는 오디오·비디오 코덱이 조금씩 다르므로, 가능한 여러 포맷의 `<source>`를 제공한다
3. **자동재생 주의**
    - `autoplay`를 사용할 경우, 사용자 경험을 해치지 않도록 `muted` 속성을 함께 사용하는 등 세심한 설정이 필요하다
4. **대체 텍스트와 자막**
    - 시각·청각장애인을 위해 대체 텍스트(이미지), 자막(동영상), 스크립트(오디오) 등을 꼼꼼히 작성한다

# 6. 하이퍼링크와 내비게이션

웹페이지 간 이동, 문서 내 특정 구역으로 이동 등을 가능하게 하는 하이퍼링크(`a` 태그)는 웹의 핵심 요소이다

효과적인 내비게이션 구성을 통해 사용자 경험을 향상시킬 수 있으며, 검색엔진이 사이트 구조를 이해하는 데에도 도움이 된다

## 1) 하이퍼링크(`a` 태그)의 기본

```html
<a href="https://www.example.com">Example 사이트 바로가기</a>
```

1. `href`: 연결할 대상(웹페이지, 파일, 문서 내 구역 등)의 경로 또는 URL을 지정
2. 링크 텍스트: 사용자가 클릭할 수 있는 텍스트(또는 이미지)
3. `target` 속성: 링크가 열리는 위치를 지정 (예: `_blank`, `_self` 등)

### `a` 태그의 주요 속성

- `href`: 이동할 URL 또는 앵커(문서 내 특정 id)를 지정
- `target="_blank"`: 새 탭 혹은 새 창에서 링크를 열기
- `target="_self"`: 현재 창(기본값)
- `rel="noopener noreferrer"`: 보안 및 성능 측면에서 새 창을 열 때 함께 사용 권장
- `download`: 파일 다운로드 링크로 사용 가능(파일명 지정도 가능)

### 문서 내 구역으로 이동하기(앵커 링크)

```html
<a href="#section1">섹션 1로 이동</a>

<section id="section1">
  <h2>섹션 1</h2>
  <p>여기가 섹션 1의 내용</p>
</section>
```

- 이동하려는 영역에 `id`를 부여하고, `href` 속성을 `#id명`으로 지정
- 링크 클릭 시 해당 구역으로 스크롤되어 이동
